Pipeline

How It Works

Three stages transform a student's raw text into an explainable, defensible score.

⚓ 1

Anchor Extraction

KeyBERT extracts the top 5 key concepts (anchors) from the reference answer using TF-IDF scored keyphrases — unigrams, bigrams, and trigrams.

🗺️ 2

Semantic Mapping

The student's answer is compared against each anchor via semantic similarity (sentence embeddings), Jaccard word overlap, and edit-distance phrasing match.

📊 3

Score + Explain

A Random Forest model (trained on the Mohler dataset) predicts the score. SHAP values attribute each feature's contribution so every point gained or lost is explained.

🎯

Key Concept Coverage

Measures what fraction of the anchor concepts from the ideal answer appear in your response. The single most important factor.

🧬

Semantic Similarity

Captures meaning beyond exact words — your answer can use synonyms and still score well if the underlying idea matches.

📝

Vocabulary Overlap

Rewards using the correct domain-specific terminology from the model answer, which is critical in technical subjects.

About

The Research Behind It

🎓 Project Background

This system is built on the Mohler Short Answer Grading Dataset — a benchmark corpus of student responses to computer science questions, each graded on a 0–5 scale by human annotators.

The pipeline addresses the twin problems of length-bias noise and black-box scoring by anchoring the score to specific, extractable concepts rather than surface-level text statistics.

SHAP (SHapley Additive exPlanations) values are used to decompose the Random Forest prediction into per-feature contributions, ensuring every point gained or lost can be traced back to a concrete aspect of the student's answer.

📊 Feature Descriptions

Feature What It Measures RF Weight
feat_anchors_covered Fraction of key-concept anchors from the ideal answer that the student addressed (threshold 0.35 cosine) 35%
feat_avg_semantic Average cosine similarity between the student answer embedding and each anchor embedding 30%
feat_max_semantic Best (maximum) cosine similarity score across all anchor comparisons — captures the student's strongest matching phrase 18%
feat_avg_jaccard Average Jaccard word-set overlap between the student answer and each anchor — rewards correct terminology 10%
feat_avg_edit Average normalised Levenshtein similarity — captures phrasing closeness beyond word-level comparison 7%
